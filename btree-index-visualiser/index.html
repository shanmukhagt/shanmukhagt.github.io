The Problem: Finding Data is Hard

Imagine a library with 1 million books — but they're all in a giant pile with no order, no catalogue. To find "Harry Potter", you'd have to pick up every single book until you found it. That could take a very long time.

Databases have the same problem

A database might store millions of user records. When you search for a specific user, without any index the database has to read every single row — one by one — until it finds a match. This is called a full table scan and it gets painfully slow as data grows.

Real world analogy

Searching a database without an index is like trying to find a name in a phone book that isn't sorted — you read every entry from page 1.

The Solution: A Sorted, Layered Index

A B-Tree is like a well-organised library with section signs, shelf labels, and alphabetical books. Instead of checking every book, you follow the signs to jump straight to the right area.

How the tree is structured

The tree has levels. At the top is the root — it holds a few key values that act as signposts. Each signpost says "go left for smaller values, go right for larger ones." You follow pointers down until you reach a leaf node — the bottom level — which holds the actual data pointers.

[ 10 | 25 ]

Leaf nodes (actual data)
Library analogy

Root = the main hall signs (Fiction / Non-Fiction). Internal nodes = aisle signs (A–F, G–M, N–Z). Leaf nodes = the actual book shelves where you find your book.

Insertions & Node Splits

Every new document you insert has its key placed into the tree — always keeping everything sorted. But what happens when a node gets too full?

Node capacity

In this visualizer, each node can hold a maximum of 3 keys (configurable by the degree t=2). When a 4th key tries to fit in, the node splits in two — the middle key is promoted up to the parent node to act as the new divider.

1
Walk down the tree following the path where the new key belongs (highlighted in yellow)
2
If a node is full, split it before descending — the middle key moves up (shown in red then violet)
3
Insert in the leaf — the key lands in its sorted position (violet glow)
Split analogy

A bookshelf gets full. You split it into two shelves and put a new sign in the aisle saying which shelf has which range of books. The aisle sign is the "promoted key."

📈

Index Scans & Range Queries

The real power of a B-Tree index is range queries — finding all documents where age > 30, or names starting with "A". The tree lets the database skip huge portions of data instantly.

How a range scan works

The database reads the internal nodes (signposts) to identify which subtrees can possibly contain matching values. Subtrees that are entirely out of range are pruned — skipped completely. Only the relevant leaf nodes are visited to collect results.

Violet — matched: this key satisfies your filter
Faded — missed: key was visited but didn't match
Greyed out — pruned: entire subtree skipped by the index
Cyan cursor — the current position being evaluated
Database query analogy

SELECT * FROM users WHERE age > 30 — instead of reading all users, the B-Tree jumps directly to age=30 and reads forward. Everything below 30 is pruned.

How to Use This App

Follow these steps to see the B-Tree come to life:

1
Configure the Index — in the left panel, name your field (e.g. age), set its type to number, then click Create Index.
2
Insert Documents — use Manual to type values one at a time, or Bulk to auto-generate many at once. Watch the tree grow and split.
3
Try a Scan — open Filter / Sort Scan, set a filter like age > 30, and click Run Scan. See how the tree prunes irrelevant subtrees.
4
Step-by-step mode — enable Step-by-step mode in Animation Controls to pause after each event and read what's happening.
5
Try compound indexes — click + Field to add a second field (e.g. name). This creates a composite index — powerful for multi-field queries.
Pro tip — try the example datasets

Switch to the JSON tab and pick Employees, Products, or Orders from the dropdown. This loads 20 realistic documents instantly so you can explore without typing anything.
